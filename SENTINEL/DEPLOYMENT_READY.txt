   ```bash
   export LIVE_APPROVAL_PHRASE="i approve live trading witgh option A"
   export DHAN_API_KEY="your_dhan_api_key"

   export DHAN_ACCOUNT_ID="your_account_id"
   ```

2) Sanity checks before starting:
   - Ensure `dhanhq` library is installed in the active venv: `pip install dhanhq`
   - Run quick config check:
     ```bash
     python3 -c "import config; print('LIVE_ENABLED=', config.LIVE_ENABLED); print('LIVE_APPROVAL_PHRASE=', repr(config.LIVE_APPROVAL_PHRASE))"
     ```

3) Start dashboard & API (optional):
   ```bash
   # Start API (in one terminal)
   python3 api.py

   # Start dashboard dev server in another terminal (optional for local UI)
   cd dashboard
   npm run dev
   ```

4) Start bot in LIVE mode (final step):
   ```bash
   # This will still honor is_live_allowed() guard; ensure ENABLE_LIVE and phrase set
   python3 main.py --execution LIVE --mode SCALP
   ```

5) Dashboard confirmation (optional):
   If you want to set an in-memory confirmation flag for the running API (useful for operators):
   ```bash
   curl -X POST http://localhost:5000/enable_live -H 'Content-Type: application/json' \
     -d '{"approval_phrase":"i approve live trading witgh option A"}'
   ```

6) Immediate safety checklist after launch:
   - Monitor logs: `tail -f sentinel.log`
   - Verify `Execution Engine initialized - Mode: LIVE` in logs
   - Verify first orders are simulated for 1-2 minutes (expect low volume)
   - If anything unexpected occurs, trigger kill switch in dashboard or send STOP command to `/command` endpoint

7) Rollback / Emergency stop:
   - To immediately stop live order placement, unset `ENABLE_LIVE` in the environment of the running process and restart the bot. Also use the dashboard `EXIT` command to clear positions.

   Skip if trading only spot/futures contracts.
